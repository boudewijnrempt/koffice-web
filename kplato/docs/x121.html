2.2. Functional Breakdown

The following sections describe the general functionality that has been described for KPlato.

2.2.1. Change Control

KPlato should be able to support the following procedures:

  • Create work breakdown structures

  • Manage a change order mechanism

2.2.2. Schedule Planning

KPlato should be able to support the following sequence of tasks for schedule planning:

  1. Take a project and break it down into tasks

  2. Assign the relationships between those tasks

  3. Assign durations to the tasks

2.2.3. Resource Allocation

KPlato should be able to support the following sequence of tasks for resources:

  1. Assign resource to tasks

  2. Calculate resource needs over a period of time

  3. Perform cost analysis control

  4. Create cost breakdown structures

2.2.4. Statusing

KPlato should be able to support the following statusing functions:

  1. Enter percent complete on a task

  2. Calculated projected actual completion date

  3. Recalculate planning dates based on actual schedule slip

2.2.5. Wishlist Items

The following items are not expected to be implemented in the first release, but could be incorporated into subsequent releases:

  • Assign ownership to tasks

  • Networked planning functions so task owners can do their own planning and have it roll up into a master plan

  • Easily copy the project files between a laptop and a desktop

  • Incorporate permission and access controls for users, groups, and managers

  • Email notifications of significant events
